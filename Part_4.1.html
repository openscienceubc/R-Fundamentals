Matrix Data

Matrix Data

Vectors are single dimension objects.

Arrays are multidimensional. A matrix is a special case of an array that contains only 2 dimensions; they have length and width but no depth. Think of this like a an excel document that has only one sheet.

  • Vectors have one dimension
  • Arrays have two or more directions
  • Matrices are a type of array that has only 2 dimensions

Right, using our script, let’s generate some data to play with.

Note

Generating all of our own data may seem a little tedious, but it’s both a great way to troubleshoot, get familiar with a new concept, and once you move into online forums for support, this support will often use ad hoc generated data to provide a solution, so it’s good to be familiar with the processes invovled in data generation.

Let’s say we’re collecting data on people, including their height and weight. To simulate a random sample let’s create the following vectors:

Height <- sample(120:190, 20, replace = TRUE) # A random sample of 20 data points from the range 120 to 190 where values can be replaced

Weight <- sample(50:90, 20, replace = TRUE) # A random sample of 20 data points from the range 50 to 90 where values can be replaced

Height
##  [1] 147 162 166 159 124 127 151 161 180 174 127 153 122 158 130 142 162 165 171 135
Weight
##  [1] 69 74 61 73 55 78 84 76 80 84 77 68 86 61 52 71 90 84 68 60

Combining Data

We now have two uni-dimensional objects; two vectors.

Let’s convert these into a two dimensional array, or matrix by combining them together into a single object that we will call myMatrix, because we’re creative that way.

cbind()

To do this, we will use the function cbind, as in column bind; a handy function that allows us to append one vector of data onto another, which will give us a matrix with 2 variables each with 20 data points.

We will give cbind() two arguments, the variable names of our two vectors.

myMatrix <- cbind(Height, Weight)

head(myMatrix)
##      Height Weight
## [1,]    147     69
## [2,]    162     74
## [3,]    166     61
## [4,]    159     73
## [5,]    124     55
## [6,]    127     78

Great, now we have a gird of data.

Just to double check that we’ve done what we intended, let’s ask R about the class of our new object:

class(myMatrix)
## [1] "matrix" "array"

Note

The length of each variable, that is each vector, in a matrix must be the same length. The above example works well because both Height and Weight have the same length.

Also, matrices, like vectors, are atomic, that is, an array can only contain one data type - numeric, character, logical etc. We cannot have a matrix where one vector is numeric data and the second is character data. We have other options that we’ll look at for this in a moment.

Let’s now create a matrix, with three columns to see a few more things that we can do with matrices and arrays.

And let’s now say that we are also interested in collecting age in addition to height and weight. So, as before, let’s create a sample of 20 ages from an undergraduate population, so, between 20 and 25:

Age <- sample(20:25, 20, replace = TRUE) # A random sample of 20 data points from the range 20 to 25 where values can be replaced

Age
##  [1] 24 24 20 23 25 24 23 23 22 24 25 25 22 22 20 25 23 21 21 22

We’re going to employ the function cbind() again to join this new variable and associated data with our existing matrix.

myMatrix <- cbind(myMatrix, Age) 

head(myMatrix)
##      Height Weight Age
## [1,]    147     69  24
## [2,]    162     74  24
## [3,]    166     61  20
## [4,]    159     73  23
## [5,]    124     55  25
## [6,]    127     78  24

We now have three columns, one for Height, Weight and Age respectively. And we’re finally starting to see something like a proper data set!

rbind()

In the same way that we can add columns to our data, we can also add rows to our data. We are not limited to adding a single column or a single row at a time.

In the following steps, we will use rbind(), as in row bind, to merge two datasets together.

Say we get two datasets of height, weight, and age data, each collected by two different researchers. We already have one set myMatrix. Let’s create a second dataset, yourMatrix to work through this.

As before, we’ll take two samples and cbind them together.

Height2 <- sample(120:190, 20, replace = TRUE)

Weight2 <- sample(50:90, 20, replace = TRUE)

Age2 <- sample(20:25, 20, replace = TRUE)

Height2
##  [1] 136 143 164 138 137 132 134 176 187 178 135 169 181 147 170 186 134 165 169 169
Weight2
##  [1] 73 70 69 66 86 79 78 63 73 74 84 58 71 82 89 66 78 74 55 89
Age2
##  [1] 25 23 22 20 25 23 22 24 24 23 20 20 24 24 21 22 23 25 21 22
yourMatrix <- cbind(Height2, Weight2, Age2)

head(yourMatrix)

## [1,]     136      73   25
## [2,]     143      70   23
## [3,]     164      69   22
## [4,]     138      66   20

Now, let’s put them together into a new matrix called completeMatric.

completeMatrix <- rbind(myMatrix, yourMatrix)

Let’s use View() to see if we were successful

Atomic data

Remember we said that matrices, like vectors, are atomic. Let’s see what the implications of this are.

Imagine we were also collecting eye colour as a variable. What we’ll do then is create a vector of 20 random samples of eye colour and add it as a fourth column to our matrix, named this time, myMatrixEyeColour.

Remember, we already created a vector of eye colours from which we can generate a random sample.

eyeColour
## [1] "blue"  "green" "brown" "hazel" "amber" "grey"

Let’s use this to generate a sample of 40 data points that we can add to our completeMatrix

EyeColourSample <- sample(eyeColour, 40, replace = TRUE) # A random sample of 40 data points from the character options in the variable eyeColour where values can be replaced

EyeColourSample
##  [1] "brown" "amber" "green" "brown" "grey"  "green" "hazel" "green" "brown" "blue"  "grey"  "hazel" "amber"
## [14] "grey"  "brown" "brown" "amber" "blue"  "blue"  "green" "brown" "amber" "amber" "hazel" "hazel" "hazel"
## [27] "blue"  "amber" "grey"  "grey"  "brown" "brown" "green" "green" "grey"  "grey"  "blue"  "hazel" "grey" 
## [40] "green"

Now, we’ll add this to completeMatrix with cbind()

completeMatrixEyeColour <- cbind(completeMatrix, EyeColourSample)

head(completeMatrixEyeColour)
##      Height Weight Age  EyeColourSample
## [1,] "147"  "69"   "24" "brown"        
## [2,] "162"  "74"   "24" "amber"        
## [3,] "166"  "61"   "20" "green"        
## [4,] "159"  "73"   "23" "brown"        
## [5,] "124"  "55"   "25" "grey"         
## [6,] "127"  "78"   "24" "green"

What’s happened? R has adapted on the fly to allow you to do this.

Inadvertently, however, it has converted all of your numerical data to character data! Remember we said that R is a little too easy going and let's you do things it probably shouldn’t? This is one case in point. You should always be careful and always inquire to see if what you intended to have happen actually happened.

Indexing

Matrices, like vectors are indexed, but now they have two axes that are indexed, rows and columns.

We can work with these just as we did with vectors, but we now need to specify if we’re interested in a row or a column of data - that is, a variable or an observation.

We do this in the following way: matrix[row, column].

So, say we wanted to see only the Height data. We could can do this in one of two ways, with an index number or an index label:

completeMatrix[, 1]
##  [1] 147 162 166 159 124 127 151 161 180 174 127 153 122 158 130 142 162 165 171 135 136 143 164 138 137 132 134
## [28] 176 187 178 135 169 181 147 170 186 134 165 169 169
completeMatrix[, "Height"]
##  [1] 147 162 166 159 124 127 151 161 180 174 127 153 122 158 130 142 162 165 171 135 136 143 164 138 137 132 134
## [28] 176 187 178 135 169 181 147 170 186 134 165 169 169

If we wanted to see Height and Age only, we can concatenate just as we did with our vector:

head(completeMatrix[, c(1,3)])
##      Height Age
## [1,]    147  24
## [2,]    162  24
## [3,]    166  20
## [4,]    159  23
## [5,]    124  25
## [6,]    127  24
head(completeMatrix[, c("Height", "Age")])
##      Height Age
## [1,]    147  24
## [2,]    162  24
## [3,]    166  20
## [4,]    159  23
## [5,]    124  25
## [6,]    127  24

To see a row:

completeMatrix[1, ]
## Height Weight    Age 
##    147     69     24
completeMatrix[1:3, ]
##      Height Weight Age
## [1,]    147     69  24
## [2,]    162     74  24
## [3,]    166     61  20

Note

By not indicating what row or column we’re interested in, R defaults to showing us the entire row or column.

To see an intersection:

completeMatrix[1,3]
## Age 
##  24
completeMatrix[1, "Age"]
## Age 
##  24
completeMatrix[1:3, c("Height", "Age")]
##      Height Age
## [1,]    147  24
## [2,]    162  24
## [3,]    166  20

Questions

Three questions that we can ask of multi-dimensional structures in R that can come in handy include.

  • nrow() which tells us the number of rows
  • ncol() which tells us the number of columns
  • dim() which tells us both pieces of information
nrow(completeMatrix)
## [1] 40
ncol(completeMatrix)
## [1] 3
dim(completeMatrix)
## [1] 40  3

What have we learned

  • We can store 2 dimensional data in a matrix
  • A matrix is atomic, that is, it can only hold one data type
  • We can combine vectors to make a matrix with cbind()
  • We can combine datasets together with rbind()
  • The vectors of an array need be of the same length
